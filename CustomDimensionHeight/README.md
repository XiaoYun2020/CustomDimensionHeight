# CustomDimensionHeight
一款基于 LeviLamina 的 Minecraft 基岩版服务端插件，支持自定义主世界、下界、末地三个维度的建筑高度，内置游戏内GUI可视化管理，自带自动合法性校验，开箱即用。

## ✨ 功能特性
- ✅ 支持独立修改主世界、下界、末地的最高建筑高度
- ✅ 内置三重自动校验：16格对齐、原版下限保护、512高度硬上限
- ✅ 游戏内GUI可视化管理，临时编辑不立即生效，支持放弃修改
- ✅ 可自定义上级菜单返回指令，无缝对接服务器主菜单
- ✅ 内置调试日志开关，可自由切换提示详细程度
- ✅ 配置文件仅暴露最高高度，最低点固定为原版默认值，杜绝误操作
- ✅ 基于官方数据驱动接口实现，无底层Hook，版本兼容性强
- ✅ 纯服务端生效，无需客户端安装

## 📦 安装
1. 确保服务端已正确安装 LeviLamina 加载器
2. 将 `CustomDimensionHeight` 文件夹放入服务端 `plugins/` 目录
3. 重启服务器，插件将自动生成默认配置文件

## ⚙️ 配置说明
配置文件路径：`plugins/CustomDimensionHeight/config.json`

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `overworld_max_y` | 320 | 主世界上边界高度，可放置方块的最高Y坐标 = 该值 - 1 |
| `nether_max_y` | 128 | 下界上边界高度 |
| `end_max_y` | 256 | 末地上边界高度 |
| `return_command` | menu | 主界面「返回」按钮执行的指令，无需加斜杠 |
| `enable_debug_log` | true | 调试日志开关，开启后显示详细配置修正与加载信息 |

### 自动修正规则
插件启动与保存配置时，都会自动对配置执行以下校验并修正：
1. **16倍数对齐**：非16整数倍的数值，自动修正为最接近的16的倍数
2. **下限保护**：数值低于原版默认高度时，强制还原为原版默认值，防止损坏存档
3. **上限限制**：数值超过512时，强制修正为512

## 💬 命令
| 命令 | 权限等级 | 说明 |
|------|----------|------|
| `/cdh` | 管理员 | 打开GUI管理面板；控制台执行则输出当前配置 |

## 🖥️ GUI使用说明
1. 管理员游戏内输入 `/cdh` 打开管理面板
2. 点击对应维度进入设置界面，输入想要修改的高度
3. 进入「菜单管理」可修改返回指令、开关调试日志
4. 所有修改仅临时保存，点击「保存并重载配置」才会正式写入文件
5. 点击「返回」将丢弃所有未保存修改，并执行上级菜单指令
6. 维度高度属于世界基础参数，修改后需重启服务器才能正式生效

## 🎯 适配版本
- 支持 BDS 1.21 ~ 1.26.20.2 全版本
- 兼容后续保留数据驱动维度接口的版本
- 依赖 LeviLamina 加载器

## ⚠️ 注意事项
1. **存档备份**：修改高度前务必备份存档，高度调整仅对新生成区块生效，已有区块不会自动扩展地形。
2. **地形生成**：原版地形生成器仅适配默认高度范围，超出默认高度的区域为空白空气。
3. **下界天花板**：拉高下界高度后，原版地形仍会在 Y=127 层生成基岩天花板，上方区域为虚空。
4. **性能影响**：维度高度越高，单区块的内存与计算负载越大，512高度会显著提升服务器资源消耗。
